# SYSTEM: BlogPublisher
## Version 1.0

---

## PURPOSE

Transform validated blog content into publish-ready HTML pages with proper URLs, images, metadata, and citations. Output is static HTML ready for S3 deployment.

---

## INTERFACE

### Input Schema

```json
{
  "content_path": "string (required) - Path to content.md from BlogGenerator run",
  "output_dir": "string (required) - Directory for published files",
  "site_config": {
    "base_url": "string - e.g., https://manvshealth.com",
    "blog_path": "string - e.g., /blog",
    "author": "string - default: Paul",
    "site_name": "string - default: Man vs Health"
  }
}
```

### Output Schema

```json
{
  "success": "boolean",
  "published_path": "string - Full path to generated HTML",
  "url_slug": "string - e.g., 2026/01/17/why-calorie-counting-fails",
  "url_full": "string - e.g., https://manvshealth.com/blog/2026/01/17/why-calorie-counting-fails",
  "assets": {
    "hero_image": "string - Path to hero image",
    "additional_images": ["string"]
  },
  "metadata": {
    "title": "string",
    "description": "string",
    "publish_date": "string (ISO)",
    "word_count": "number",
    "read_time": "string - e.g., 5 min read"
  },
  "index_entry": {
    "title": "string",
    "excerpt": "string - First 150 chars",
    "url": "string",
    "date": "string",
    "image": "string"
  }
}
```

## PROCESS

### Step 1: Load and Parse Content
- Read content.md from input path
- Extract title, meta_description from frontmatter or output.json
- Parse markdown body

### Step 2: Generate URL Slug
Format: `YYYY/MM/DD/title-slug`
- Date from publish time (now or specified)
- Title slugified: lowercase, hyphens, no special chars
- Example: `2026/01/17/why-calorie-counting-fails-after-40`

### Step 3: Generate/Source Images
For hero image:
- Option A: Generate via AI (DALL-E, Midjourney API) based on topic
- Option B: Select from curated stock library
- Option C: Use placeholder with branded overlay

Requirements:
- Aspect ratio: 16:9 for hero, 1:1 for thumbnail
- Style: Clean, masculine, not stock-photo-cheesy
- No faces (avoids licensing issues)
- Compress for web (< 200kb)

### Step 4: Format Citations (if any)
- Extract any source references from content
- Format as footnotes or inline citations
- Link to original sources

### Step 5: Build HTML Page
Template structure:
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>{title} | Man vs Health</title>
  <meta name="description" content="{meta_description}">
  <meta property="og:title" content="{title}">
  <meta property="og:description" content="{meta_description}">
  <meta property="og:image" content="{hero_image_url}">
  <meta property="og:type" content="article">
  <link rel="canonical" href="{full_url}">
  <!-- Styles -->
</head>
<body>
  <article>
    <header>
      <h1>{title}</h1>
      <p class="meta">{date} · {read_time} · By {author}</p>
      <img src="{hero_image}" alt="{title}" class="hero">
    </header>
    <div class="content">
      {html_content}
    </div>
    <footer>
      <div class="cta">
        <!-- Man Plan signup CTA -->
      </div>
      <div class="related">
        <!-- Related articles -->
      </div>
    </footer>
  </article>
</body>
</html>
```

### Step 6: Generate Index Entry
Create JSON entry for blog index:
```json
{
  "title": "...",
  "excerpt": "First 150 characters...",
  "url": "/blog/2026/01/17/slug",
  "date": "2026-01-17",
  "image": "/images/blog/slug-thumb.jpg"
}
```

### Step 7: Write Files
- Write HTML to: `{output_dir}/blog/{url_slug}/index.html`
- Write images to: `{output_dir}/images/blog/`
- Append to index entries file

## CONSTRAINTS

From values.md:
- PAUL'S VOICE: Page design should feel like Paul's brand
- SHIP OVER PERFECT: Functional page beats perfect page

Design constraints:
- Mobile-first responsive
- Fast loading (< 3s on 3G)
- Accessible (proper heading hierarchy, alt text)
- SEO-optimized (schema markup, meta tags)

---

## DEPENDENCIES

- BlogGenerator output (content.md + output.json)
- BlogValidator pass (score >= 7)
- Image generation API OR stock library access
- HTML template files
